# 1.2. IoC/DI를 위한 빈 메타정보 작성
## 1. 빈 설정 메타정보
스프링은 `BeanDefinition`타입의 오브젝트를 메타정보로 활용하여 Bean을 관리한다.

### 1.1. `BeanDefinition` 특징
* 가장 중요한 정보는 `beanClassName`이다.
* `BeanDifinition`과 Bean은 1:N 관계일 수 있다.
  * 따라서 `BeanDifinition`에 Bean의 ID나 객체정보가 들어있지 않다.

## 2. 빈 등록 방법
### 2.1. XML:<bean>태그
```xml
<bean id="hello" class="spring.test.hello">
    ...
</bean>
```

### 2.2. XML: 네임스페이스와 전용 태그
```xml
<aop:pointcut id="mypointcut" expression="execution(* *..*ServiceImpl.upgrade*(..))" />
```
* 기술적인 설정과 기반 서비스를 빈으로 등록할 때 의미가 더 잘 드러나도록 네임스페이스와 태그를 가진 설정 방법을 제공

### 2.3. 자동인식을 이용한 빈등록: 스테레오타입 애노테이션과 빈스캐너
#### 1) 스테레오타입 애너테이션
* 스프링의 빈 스캐너의 디폴트 필터에 적용되는 애너테이션
* `@Component`, `@Service`, `@Repository`, ...
* 기본적으로 빈 이름은 클래스의 이름의 가장 앞 부분을 소문자로 변경한 뒤 사용한다.

#### 2) 빈 스캐너 등록 방법
1. XML 이용
    ```xml
    <context:component-scan 
        base-package="com.example"/>    
    ```
2. 빈스캐너를 내장한 애플리케이션 컨텍스트 사용
* servlet-context 설정에 contextClass에 `AnnotationConfingWebAnnotationContext` 지정

> 📌 스프링 부트에 어떻게 대응되는지
```java
@Configuration
@ComponentScan(basePackages = "com.example")
public class AppConfig {
}
```
* XML의 Component Scan 태그는 해당 어노테이션과 매핑된다.
```java
@SpringBootApplication
public class TechQBankApplication {
    public static void main(String[] args) {
        SpringApplication.run(TechQBankApplication.class, args);
    }
}
```
* 스프링 부트에는 메인 클래스에 `@SpringBootApplication`이 붙어있는데 이는 세가지 어노테이션의 역할을 한다.
```java
@SpringBootConfiguration
@EnableAutoConfiguration
@ComponentScan 
```
| 어노테이션                      | 역할                                                                            |
| -------------------------- | ----------------------------------------------------------------------------- |
| `@ComponentScan`           | 내 프로젝트 코드에서 `@Component`, `@Service`, `@Repository`, `@Controller` 등을 찾아 빈 등록 |
| `@EnableAutoConfiguration` | classpath, 설정값, 조건을 보고 스프링 부트가 필요한 설정 빈을 자동 등록                                |
| `@SpringBootConfiguration` | 현재 클래스가 설정 클래스임을 나타냄. `@Configuration`의 스프링 부트 버전                             |

### 2.4. 자바 코드에 의한 빈 등록: @Configuration, @Bean
```java
@Configuration
public class TestConfig {
    @Bean
    public Hello hello(){
        return new Hello();
    }
}
```
* `@Configuration` 클래스는 자동스캔 방식으로 Bean으로 등록된다.
* `@Configuration` 내부에 `@Bean` 메서드는 호출하더라도 기존의 Bean이 반환된다.
  * Proxy로 동작하기 때문 - Vol1 다은님 설명 참고

### 2.5. 자바 코드에 의한 빈 등록: 일반 빈 클래스의 @Bean 메서드
```java
public class TestService {
    @Bean
    private Hello hello(){
        return new Hello();
    }
}
```
* Bean으로 등록되나 @Bean` 메서드는 호출할 때마다 새로운 객체가 생성된다.
* 따라서 외부에서 호출 불가능하도록으로 선언해두고 클래스 내부에서도 DI를 통해 참조해야한다.
* 사용하는 상황
  * 일반적으로 @Bean 메서드를 통해 정의되는 빈이 클래스로 만들어지는 빈과 매우 밀접한 관계가 있는 경우, 특별히 종속적인 빈인 경우에 사용할 수 있다.
  * Bean 메서드는 클래스 내부에 정의되어 있으므로 모든 정보에 접근 가능하여 설정정보를 공유할 수 있고, 외부로 빈의 존재가 노출되지 않으므로 빈의 존재를 감출 수 있다.

## 3. 의존관계 주입 방법
### 3.1. XML: <property>, <constructor-arg>
```xml
<!-- 수정자 주입-->
<bean ...>
    <property name="printer" ref="defaultPrinter"/>
</bean>
<bean id="defaultPrinter" class="..." />
```
```xml
<!-- 생성자 주입-->
<bean ...>
    <constructor-arg index="0" ref="defaultPrinter"/>
</bean>
<bean id="defaultPrinter" class="..." />
```

### 3.2. XML: 자동와이어링
```xml
<!-- byName : 빈 이름 자동 와이어링-->
<bean id="hello" class="...Hello" autowire="byName"> 
    <!-- <property name="printer" ref="printer"/> 생략-->
</bean>
```
```xml
<!-- byType : 빈 타입 자동 와이어링-->
<bean id="hello" class="...Hello" autowire="byType"> 
    <!-- <property name="printer" ref="printer"/> 생략-->
</bean>
```
* 단 타입이 같은 여러개의 빈이 존재할 때는 사용 불가능
* 타입이 이름보다 느리다.
* 의존관계를 한눈에 파악할 수 없다는 단점 존재.

### 3.3. XML: 네임스페이스와 전용 태그
* 관례적으로 전용 태그에 의해 만들어지는 빈을 다른 빈이 참조할 경우 id애트리뷰트를 사용해 빈의 아이디를 지정한다
* 자세한건 알아서 찾아보기

### 3.4. 애노테이션: @Autowired
* "타입" 기준 주입
* 문제: 같은 타입 빈이 여러개일 때
  * `NoUniqueBeanDefinitionException` 오류 발생

### 3.5. 애노테이션: @Qualifier
* @Qualifier는 단독으로 의존성을 주입하는 어노테이션이 아니라, 보통 @Autowired와 함께 사용해서 어떤 빈을 주입할지 지정
```java
@Component
@Qualifier("rateDiscountPolicy")
public class RateDiscountPolicy implements DiscountPolicy {
} 
```
```java
@Component
@Qualifier("fixDiscountPolicy")
public class FixDiscountPolicy implements DiscountPolicy {
}
```
```java
@Service
public class OrderService {

    private final DiscountPolicy discountPolicy;

    @Autowired
    public OrderService(@Qualifier("rateDiscountPolicy") DiscountPolicy discountPolicy) {
        this.discountPolicy = discountPolicy;
    }
}
```

### 3.6. 애노테이션: @Resource
* 스프링 전용이 아니라 자바 표준 계열 어노테이션
* "이름"기준 주입
```java
import jakarta.annotation.Resource;

@Service
public class OrderService {

    @Resource(name = "rateDiscountPolicy")
    private DiscountPolicy discountPolicy;
} 
```
| 구분     | `@Autowired` | `@Qualifier`        | `@Resource`        |
| ------ | ------------ | ------------------- | ------------------ |
| 소속     | Spring       | Spring              | Jakarta/Java 표준 계열 |
| 주 역할   | 의존성 주입       | 주입할 빈 지정 보조         | 의존성 주입             |
| 기본 기준  | 타입           | 이름 또는 qualifier 값   | 이름                 |
| 단독 사용  | 가능           | 보통 단독 사용 X          | 가능                 |
| 생성자 주입 | 적합           | `@Autowired`와 함께 사용 | 부적합                |
| 필드 주입  | 가능           | 가능                  | 가능                 |
| 실무 권장도 | 높음           | 여러 빈 구분 시 사용        | 상대적으로 낮음           |


### 3.7. 자바코드에 의한 의존관계 설정: @Bean 메소드 호출
```java
@Configuration
public class TestConfig {
    @Bean
    public Hello hello(){
        Hello hello = new Hello();
        hello.setPrinter(printer());
        return hello;
    }

    @Bean
    public Printer printer() {
        return new Printer();
    }
}
```

### 3.8. 자바코드에 의한 의존관계 설정: @Bean과 메소드 자동와이어링
```java
@Configuration
public class TestConfig {
    @Bean
    public Hello hello(Printer printer){
        Hello hello = new Hello();
        hello.setPrinter(printer);
        return hello;
    }

    @Bean
    public Printer printer() {
        return new Printer();
    }
}
```
* `@Bean`이 붙은 메소드는 기본적으로 `@Autowired`가 붙은 메소드처럼 동작한다.

## 4. 프로퍼티 값 설정 방법
### 4.1. XML: <property>와 전용 태그
```xml
<bean ...>
    <property name="printer" value="everyone"/>
</bean>
```
### 4.2. 애노테이션: @Value
> 책에서는 XML 기준으로 소개되어있으나 properties 기준으로 바꿈
#### 1) 기본 사용법
```java
@Component
public class AppInfo {

    @Value("${app.name}")
    private String appName;

    @Value("${app.version}")
    private String appVersion;
}
```

#### 2) 생성자 주입도 가능
```java
@Component
public class AppInfo {

    private final String appName;
    private final String appVersion;

    public AppInfo(
            @Value("${app.name}") String appName,
            @Value("${app.version}") String appVersion
    ) {
        this.appName = appName;
        this.appVersion = appVersion;
    }
}
```

#### 3) 기본값 지정 가능
```java
@Value("${app.name:Default App}")
private String appName;
```

#### 4) 타입 변환 가능
```properties
app.timeout=30
app.enabled=true
app.servers=server1,server2,server3
```
```java
@Component
public class AppConfigValue {

    @Value("${app.timeout}")
    private int timeout;

    @Value("${app.enabled}")
    private boolean enabled;

    @Value("${app.servers}")
    private String[] servers;
}
```

#### 5) `${...}`와 `#{...}`의 차이
* `${...}`: properties나 yml에 있는 설정값을 가져온다.
* `#{...}`: 스프링 표현식, 즉 SpEL을 실행한다.

```java
@Value("#{10 + 20}")
private int number;
```
* 30 주입

```java
@Value("#{userService.defaultName}")
private String defaultName;
```
* 다른 빈의 값 주입

## 5. 컨테이너가 자동 등록하는 빈
| 항목                          | 정체                  | 주요 역할                         |
| --------------------------- | ------------------- | ----------------------------- |
| `ApplicationContext`        | 스프링 컨테이너            | 빈 관리, 이벤트, 리소스, 환경 등 전체 기능 제공 |
| `BeanFactory`               | 빈 관리 컨테이너의 핵심 인터페이스 | 빈 생성, 조회, 의존성 주입              |
| `ResourceLoader`            | 리소스 로딩 인터페이스        | classpath, file, URL 리소스 읽기   |
| `ApplicationEventPublisher` | 이벤트 발행 인터페이스        | 애플리케이션 이벤트 발행                 |
| `systemProperties`          | JVM 시스템 프로퍼티        | `System.getProperties()` 값 접근 |
| `systemEnvironment`         | OS 환경변수             | `System.getenv()` 값 접근        |
